Create an account

When you sign up for an account on GitHub.com, we host your files and your organization becomes a part of the world's largest open source community. You create both public and private repositories (projects) on GitHub.com. Public repositories are always free, see our plans for private repositories.

Some users have both a free GitHub.com account for public repositories and a GitHub Enterprise account. GitHub Enterprise gives users who wish to host files on their own servers to do so and run the same GitHub experience on a local network.

1. Review the Terms

It may sound like a boring first step, but many government agencies can't just simply click sign up like you would if you were at home. The good news is we've worked out a government friendly addendum to our standard terms of service. Simply have your general council's office give it a quick once over, and you should be good to go!

2. Sign Up

The hard work is already done. Creating a GitHub organization happens in two steps. First, if you don't have one already, create a GitHub user account by visiting github.com and picking a username and password. The username should be personal to you, just like any other social network. Second, pick a name that represents your agency, and create an organization.

3. Dive in

You're all set. Now comes the fun stuff. Create your first repository, fork someone else's code, and most importantly be social, after all, GitHub's about building communities. It may be helpful to install GitHub for Windows or Mac, or if you feel comfortable dive into the command line. Welcome to GitHub.
